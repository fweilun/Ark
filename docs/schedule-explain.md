# 預約訂單與行事曆派單（MVP）

## 目標
1. 乘客可在行事曆選時間建立預約訂單。
2. 司機可提前領取預約單，越接近預約時間獎勵越高。
3. 無法承諾一定有司機，但要讓乘客提前知道風險與可能的替代方案。

## 派單方式選擇
結論：司機搶單更適合 MVP，搭配「平台提醒 + 近截止加價」。

理由：
1. 實作成本低，流程直觀，司機端只要「看得到就能接」。
2. 可用獎勵與加價來解決低供給問題，先驗證需求。
3. 方便擴展到之後的「平台派單 + 搶單混合」模式。

## 基本規則（MVP）
1. 預約單建立後進入「可被司機領取」狀態。
2. 司機可在任意提前時間領取。
3. 越接近預約時間，系統自動提高獎勵或加價，提升被領取機率。
4. 若到達「截止時間」仍無司機領取，通知乘客改為即時單或取消。

## 取消與違約政策（MVP 方向）
1. 乘客取消
   - 在取消截止時間前：免費取消。
   - 超過截止時間：可收取消費或部分補貼司機與平台。
2. 司機取消
   - 若已領取後取消：記錄違約，降低司機信用分或暫時限制搶單。
   - 系統立刻將訂單重新開放，並提高獎勵。

## 價格與獎勵策略
1. 基礎費率依照原定估價。
2. 接近預約時間的訂單提高「司機獎勵」或「平台補貼」。
3. MVP 先由平台承擔補貼成本，驗證供給與需求可行性。

## 失敗處理（無法配到司機）
1. 在截止時間前 X 分鐘提醒乘客「尚未配到」。
2. 提供三個選擇：
   - 轉即時單（可能加價）
   - 延後時間
   - 取消訂單

## 後端需要支援的資訊（文字描述）
1. 預約時間與建立時間。
2. 是否為預約單與目前狀態。
3. 司機領取時間與領取者身分。
4. 取消截止時間與實際取消時間。
5. 動態獎勵或加價設定。

## 後續可升級方向
1. 混合派單：先搶單，接近時間平台主動指派。
2. 司機排班：司機提前鎖定可服務時段。
3. 信用與風控：司機取消率、乘客取消率分級。
